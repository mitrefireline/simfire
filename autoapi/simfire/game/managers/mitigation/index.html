class simfire.game.managers.mitigation.ControlLineManager(size, pixel_scale, terrain, headless=False)#

Base class to create and manage control lines and allow for the creation of more control lines while the game is running. Child classes will change the line_type, sprite_type, and add the initial points with `

Call update() to add points.

Initialize the class with the display size of each ControlLine sprite, the pixel_scale, and the Terrain that the `ControlLine`s will be placed.

Parameters:

  • size (int) – The display size of each ControlLine point.

  • pixel_scale (float) – The amount of ft each pixel represents. This is needed to track how much a fire has burned at a certain location since it may take more than one update for a pixel/location to catch on fire depending on the rate of spread.

  • terrain (simfire.game.sprites.Terrain) – The Terrain that describes the simulation/game

  • headless (bool) – Flag to run in a headless state. This will allow PyGame objects to not be initialized.

Manages the placement of FireLines and FireLine sprites. Should have varying physical characteristics from ScratchLines and WetLines.

  • terrain (simfire.game.sprites.Terrain) – The Terrain that describes the simulation/game
